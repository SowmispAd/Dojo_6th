How many loops? ``` 1 loop → O(n) 2 loops → O(n²) 3 loops → O(n³) ``` ---------------------------------------- ### 2. Does the input get cut in half? ```python while low <= high: ``` or ```python mid = (low + high) // 2 ``` → O(log n) ---------------------------------------- ### 3. Is there sorting? ```python nums.sort() sorted(nums) ``` → O(n log n) ---------------------------------------- ### 4. Does every element enter and leave only once? Examples: * Sliding Window * Two Pointers * Monotonic Stack * BFS * DFS → O(n) ---------------------------------------- ### 5. Is every recursive call branching? ```python dfs(i + 1) dfs(i + 2) ``` → Usually Exponential ---------------------------------------- # Super Easy Space Complexity Trick Ignore: * Input array * Output required by the problem Only count the extra memory. ### Count these Variables only → O(1) ---------------------------------------- One list of size `n` → O(n) ---------------------------------------- Dictionary/Set storing `n` elements → O(n) ---------------------------------------- Stack holding `n` elements → O(n) ---------------------------------------- Queue holding `n` elements → O(n) ---------------------------------------- Matrix `n × n` → O(n²) ---------------------------------------- Recursive depth = `n` → O(n) ---------------------------------------- Recursive depth = `log n` → O(log n) ---------------------------------------- # Pattern Recognition (Most Important) | If you read this in a problem... | Think of... | Complexity | | --------------------------------- | ---------------------------- | --------------------- | | "Subarray" | Sliding Window / Prefix Sum | O(n) | | "Sorted array" | Binary Search / Two Pointers | O(log n) or O(n) | | "Next Greater/Smaller" | Monotonic Stack | O(n) | | "Top K" | Heap | O(n log k) | | "Shortest Path" | BFS / Dijkstra | O(V + E) / O(E log V) | | "Connected Components" | DFS / BFS / Union-Find | O(V + E) | | "Generate all subsets" | Backtracking | O(2ⁿ) | | "Generate all permutations" | Backtracking | O(n!) | | "Maximum/Minimum in every window" | Monotonic Queue | O(n) | | "Repeated fast lookup" | HashMap/HashSet | O(1) average lookup | ---------------------------------------- # Golden Rules (Works for 90% of DSA Problems) * Each element processed once → O(n) * Each element processed twice → Still O(n) * Nested loops over all elements → O(n²) * Input size halves each step → O(log n) * Sorting is involved → O(n log n) * Try all combinations/subsets → Exponential * Extra array/set/map/stack of size `n` → O(n) space * Only a few variables → O(1) space
